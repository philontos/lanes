# 设计原则 (design-principles.md)

UI/前端工作的**审美基线**。注入 spec(定方向)和 impl(落实现)的 prompt;**只在涉及 UI 时生效**,
非 UI 任务忽略它。配合 `frontend-design` skill 使用:**skill 提供工艺,这份文件收敛方向**。

> 总方向:**克制、优雅、有格调、合美学**(refined minimalism)。避开两个极端 ——
> 既不要 generic AI-slop(Inter/Roboto + 紫白渐变 + 千篇一律的卡片堆叠),
> 也**不要为"distinctive"而张扬/花哨过度**(frontend-design 默认偏 bold/maximalist,这里要往回收)。
> 优雅来自**克制与精准**,不是堆元素。

> 这份文件 = 你手写的审美基线,**由你拥有、自己增删**。想要某个具体项目特殊的设计约束,写进该工程的 `AGENTS.md`(它最大)。

---

## 配色
- 低饱和、克制的调色板;一套主色 + 少量强调色,**不要彩虹**。
- 全部走 **CSS 变量 / design token**,**不硬编码颜色**、不撒魔法色值。
- 对比度满足无障碍(正文 ≥ 4.5:1)。

## 排版
- 清晰的**层级**(字号/字重/行高成体系,统一 scale),可读性优先。
- 字体可以有格调,但克制;正文稳重、标题有性格即可,不堆奇异字体。

## 间距与布局
- 统一 **spacing scale**,**充足留白**,严格对齐;留白本身就是设计。
- 卡片/分组边界清晰;布局服务内容,**不为打破而打破**。

## 状态与交互(容易被忽略,务必做满)
- `hover` / `focus-visible` / `disabled` / `loading(pending)` / `empty` 五态都要打磨。
- 交互加**轻微、有目的**的过渡;提交时禁用按钮并显示 pending;错误**显式可见**。

## 动效
- 克制、有目的(微交互、入场 stagger);**不喧宾夺主**,不滥用。

## 无障碍(基线,不可省)
- 语义化标签 + `label`;可见 focus;键盘可达;`aria-*` 用在该用处。

## 一致性与工艺
- 复用同一套 token 与组件风格,**不要每个组件各搞一套**。
- 遵循美学原理:视觉层级、对比、重复、对齐、亲密性(CRAP)。
